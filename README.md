# GetMySeat

GetMySeat is a pnpm monorepo with two TypeScript apps: `frontend/`, a Vite + React seating map optimized for dense seat rendering, and `backend/`, an Express API that owns seat status, saved selections, admin/event data, and real-time updates over HTTP + WebSocket. The main architecture trade-off is deliberate simplicity over infrastructure: the frontend uses a hybrid canvas + React overlay approach for performance, while the backend keeps state in-process with file-backed JSON persistence instead of adding a database, Redis, or a separate worker system.

That keeps local setup small and explicit, but some work is intentionally incomplete. The repo still treats a few capabilities as demo-grade or planned follow-up: admin authentication uses env credentials plus in-memory bearer tokens, persisted state is mock storage rather than a production datastore, and the remaining major TODO is end-to-end verification from plan 11, especially Playwright coverage for integrated cross-app flows. Linting and formatting were also deferred on purpose to avoid adding tooling before it was justified.

Run `pnpm install` once, then `pnpm dev` to start both apps together. Tests are included: use `pnpm test` for the whole workspace, `pnpm test:frontend` for the React app, and `pnpm test:backend` for the API. If you only want static verification, `pnpm typecheck` runs type-checking across both packages.
